# 쏨킴 SPEC v1 — 예측불가 캐릭터 게임

## 0. v0 → v1 핵심 변경

**콘텐츠의 정의가 바뀜.**
- v0: 콘텐츠 = 제작된 반응 연출 (연출 개수 = 수명)
- v1: 콘텐츠 = **캐릭터의 예측불가한 머릿속** (AI 생성, 수명 무한)
- 도구는 무기가 아니라 캐릭터 속을 떠보는 **질문**. 하드코딩 연출은 타격감 담당, AI는 재미 담당으로 역할 분리

## 1. 한 줄 컨셉

**"괴롭히면 빡치고, 달래면 녹고, 가끔... 소름 돋는다."**
겉은 말랑한 캐주얼, 속은 블랙코미디. 짱구 감성 + 한 끗 어른용.

### 캐릭터 명명 규칙
- 정식 명칭: **쏨킴** (본명 김솜) / 영문 SSOMKIM
- 호칭 3모드: 평소 "쏨킴"(능청 스타일링) ↔ 진지/혼낼 때 풀네임 "김솜!!" ↔ 애칭 "솜이/솜킴이"(애정·화해 연출, 아동 발음 마모형의 공식화) — UI·대사·연출 전반에서 이 전환을 개그/감정 장치로 사용
- 스토어/SEO: 검색 키워드에 변형 표기(솜킴, 솜킴이, 솜키미) 포함해 마모 트래픽까지 회수
- 세계관 확장 작명 공식: 김씨 일가 (스킨/시즌 캐릭터: 김떡, 김빵, 김이불 등)

## 2. 2-레이어 반응 아키텍처

```
[탭/도구 입력]
   ├─ L1 즉각 레이어 (0ms, 하드코딩) ── 출렁임/사운드/파티클/게이지
   └─ L2 심리 레이어 (1~2s, AI 비동기) ─ 말풍선 대사 (뒤늦게 구시렁)
```

- **L1 = 타격감.** LLM 절대 개입 금지. JSON 정의 (상태×도구×부위 → 애니메이션/사운드)
- **L2 = 영혼.** 지연이 결함이 아니라 연출 ("...방금 그거 진심이었냐?")
- L2 실패/타임아웃 시 L1 기본 대사로 폴백 → AI 죽어도 게임은 돌아감

## 3. 대사 티어 시스템

| 티어 | 빈도 | 생성 방식 | 예시 |
|---|---|---|---|
| T1 일상 | 항상 | 하드코딩 풀 + 캐시 | "구?", "그만해!!" |
| T2 상황 | 세션당 5~10회 | AI (현재 상태 주입) | "또 새총이야? 창의력 무엇?" |
| T3 기억 | 세션당 1~3회 | AI (누적 기억 주입) | "너 어제도 배만 노리더라. 취향 한결같네" |
| **T4 정색** | **~50회에 1회** | AI (기억+정색 페르소나) | "...재밌어?" / "쏘기 전에 잠깐. 오늘 회사에서 무슨 일 있었지?" |

### T4 정색 티어 규칙 (게임의 숨겨진 무기)
1. **희소성 엄수**: 발동 상한 — 세션당 1회, 일 2회. 자주 나오면 기분 나쁜 게임, 가끔 나오면 소름 짤
2. **3초 룰**: 정색 → 정적 → 반드시 개그/애교로 복귀. "...재밌어? (정적) ...나도 재밌어 ㅋ 더 해봐." 끝맛은 항상 사랑스러움. 죄책감 게임화 금지
3. 연출 동반: 화면 정적 + BGM 뮤트 + 캐릭터 정면 응시 (L1에 정색 전용 연출 1세트 하드코딩)
4. 도감 최상위 티어 등록 → 일부러 뽑으려고 괴롭히는 역설 유도. 바이럴은 귀여움이 아니라 이 순간에서 터짐

## 4. 기억 스키마 (localStorage)

```json
{
  "stats": { "tool_counts": {"poke":120,"sling":45,"tickle":12},
             "favorite_target": "belly",
             "reconcile_count": 7,
             "betrayal_count": 5 },   // 화해 직후 3초 내 재공격 횟수
  "sessions": { "last_visit": "...", "visit_hour_pattern": [23,0,1],
                "avg_intensity_trend": "up" },
  "moments": [ "2026-06-10 첫 화해", "2026-06-11 정색 T4 발동" ]
}
```

- 프롬프트 주입 전 요약 (토큰 절약): 통계 → 자연어 3줄
- `betrayal_count`가 T4의 핵심 재료: "난 네가 화해하고 또 때릴 거 알고 있었어. 그래도 안겼던 거야."
- `visit_hour_pattern` → 새벽 접속 시: "이 시간에 나 때리러 온 거야? ...뭐, 와줘서 좋긴 해."
- `avg_intensity_trend` → "너 지난주보다 세게 때려. 무슨 일 있어?"

## 5. 페르소나 프롬프트 구조

```
[고정 페르소나] 쏨킴: 얄밉고 능청, 맞아도 기죽지 않음, 츤데레.
  반말, 한 문장, 이모지 금지, 절대 비굴하지 않음
[티어 지시] T2/T3/T4별 톤 가이드 (T4만 정색 허용)
[기억 요약] 3줄
[현재 상황] 상태머신 상태 + 직전 입력 3개
[출력 제약] 20자 이내 한 문장, JSON {"line": "..."}
```

- 말벗 5종/랄랄 상담실장 페르소나 설계 노하우 직접 이식
- 금지 영역: 실제 인신공격, 자해/우울 유도성 멘트. 정색은 "꿰뚫어봄"이지 "비난"이 아님

## 6. 비용 모델

- 모델: Haiku급. 입력 ~400토큰(요약 주입) / 출력 ~30토큰
- 세션당 호출 상한 10회 → DAU 1,000 기준 일 1만 호출 ≈ 월 수만 원 이내
- T1 캐시 + 동일 상황 24h 캐시로 실호출 30~50% 절감
- 무료 운영 한계선 도달 시: 광고 리워드로 "쏨킴가 말 많아지는 1시간" 판매 — AI 비용을 그대로 수익 장치로 전환

## 7. 감정 상태머신 (v0 유지)

평온(약올림 idle) → 짜증(30%) → 분노(70%) → 삐짐(100%, 모드 전환: 달래기) → 화해(애교 연출) → 평온.
화해 직후 방심 상태 = 재공격의 죄책감 섞인 재미 = `betrayal_count` 수집 지점.

## 8. 인터랙션 MVP (v0 유지)

- 괴롭히기 3: 찌르기(탭, 부위별), 새총(드래그, matter.js), 간지럽히기(빠른 문지름)
- 달래기 2: 쓰다듬기(느린 드래그), 간식(코인)
- 드래그 속도로 간지럽히기/쓰다듬기 분기

## 9. 검증 지표 (v1 추가분 굵게)

1. 평균 세션 ≥ 3분
2. 화해 도달률 ≥ 50%
3. 24h 재방문 ≥ 20%
4. **T4 목격자의 공유/스크린샷 행동** (수동 측정: 커뮤니티 반응) — AI 레이어 가설의 직접 지표
5. **L2 대사 노출 후 세션 연장 효과** (대사 끄기 A/B 가능 구조로)

## 10. PR 분할 (one PR = one tool, end-to-end)

1. **PR-1**: 캐릭터 렌더 + 상태머신 + L1 JSON 반응 로더
2. **PR-2**: 괴롭히기 3종 + 게이지 + 삐짐/화해
3. **PR-3**: L2 AI 레이어 — 비동기 대사 큐 + 폴백 + 티어 라우터
4. **PR-4**: 기억 스키마 + 요약기 + T3/T4 발동 조건
5. **PR-5**: 코인/도감(T4 최상위)/스킨 + PWA
6. **PR-6**: 사운드 폴리싱 + 지표 로깅 + 대사 A/B 스위치

PR-1~2는 AI 없이 완결 (장난감으로 먼저 검증 가능). PR-3부터 영혼 탑재.
